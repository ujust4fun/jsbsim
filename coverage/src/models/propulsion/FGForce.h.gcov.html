      36             : encapsulate all the functionality associated with transforming those
      37             : forces from their native coord system to the body system.  This includes
      38             : computing the moments due to the difference between the point of application
      39             : and the cg.

      42             :          user *must* always pass this class the negative of beta. This is true
      43             :          because sideslip angle does not follow the right hand rule i.e. it is
      44             :          positive for aircraft nose left sideslip.  Note that use of the custom
      45             :          transform for this purpose shouldn't be necessary as it is already
      46             :          provided by SetTransform(tWindBody) and is not subject to the same

      75             : /** Utility class that aids in the conversion of forces between coordinate
      76             :     systems and calculation of moments.
      77             : <br><h3>Resolution of Applied Forces into Moments and Body Axes Components</h3>
      78             : <br><p>
      79             : All forces acting on the aircraft that cannot be considered a change in weight
      80             : need to be resolved into body axis components so that the aircraft acceleration
      81             : vectors, both translational and rotational, can be computed. Furthermore, the
      82             : moments produced by each force that does not act at a location corresponding to
      83             : the center of gravity also need to be computed. Unfortunately, the math required
      84             : to do this can be a bit messy and errors are easily introduced so the class
      85             : FGForce was created to provide these services in a consistent and reusable
      86             : manner.<br><br></p>
      87             : 
      88             : <h4>Basic usage</h4>
      89             : 
      90             : <p>FGForce requires that its users supply it with the location of the applied
      91             : force vector in JSBSim structural coordinates, the sense of each axis in that
      92             : coordinate system relative to the body system, the orientation of the vector
      93             : also relative to body coordinates and, of course, the force vector itself. With
      94             : this information it will compute both the body axis force components and the
      95             : resulting moments. Any moments inherently produced by the native system can be
      96             : supplied as well and they will be summed with those computed.</p>
      97             : 
      98             : <p>A good example for demonstrating the use of this class are the aerodynamic
      99             : forces: lift, drag, and side force and the aerodynamic moments about the pitch,
     100             : roll and yaw axes. These "native" forces and moments are computed and stored
     101             : in the FGColumnVector objects vFs and vMoments. Their native coordinate system
     102             : is often referred to as the wind system and is defined as a right-handed system
     103             : having its x-axis aligned with the relative velocity vector and pointing towards
     104             : the rear of the aircraft , the y-axis extending out the right wing, and the
     105             : z-axis directed upwards. This is different than body axes; they are defined such
     106             : that the x-axis is lies on the aircraft's roll axis and positive forward, the
     107             : y-axis is positive out the right wing, and the z-axis is positive downwards. In
     108             : this instance, JSBSim already provides the needed transform and FGForce can make
     109             : use of it by calling SetTransformType() once an object is created:</p>
     110             : 
     111             : <p><tt>FGForce fgf(FDMExec);</tt><br>
     112             : <tt>fgf.SetTransformType(tWindBody);</tt><br><br>
     113             : 
     114             : This call need only be made once for each object. The available transforms are
     115             : defined in the enumerated type TransformType and are tWindBody, tLocalBody,
     116             : tCustom, and tNone. The local-to-body transform, like the wind-to-body, also
     117             : makes use of that already available in JSBSim. tNone sets FGForce to do no
     118             : angular transform at all, and tCustom allows for modeling force vectors at
     119             : arbitrary angles relative to the body system such as that produced by propulsion
     120             : systems. Setting up and using a custom transform is covered in more detail
     121             : below.
     122             : Continuing with the example, the point of application of the aerodynamic forces,
     123             : the aerodynamic reference point in JSBSim, also needs to be set:</p>
     124             : <p><tt>
     125             : fgf.SetLocation(x, y, z)</tt></p>
     126             : 
     127             : <p>where x, y, and z are in JSBSim structural coordinates.</p>
     128             : 
     129             : <p>Initialization is complete and the FGForce object is ready to do its job. As
     130             : stated above, the lift, drag, and side force are computed and stored in the
     131             : vector vFs and need to be passed to FGForce:</p>
     132             : 
     133             : <p><tt>fgf.SetNativeForces(vFs);</tt> </p>
     134             : 
     135             : <p>The same applies to the aerodynamic pitching, rolling and yawing moments:</p>
     136             : 
     137             : <p><tt>fgf.SetNativeMoments(vMoments);</tt></p>
     138             : 
     139             : <p>Note that storing the native forces and moments outside of this class is not
     140             : strictly necessary, overloaded SetNativeForces() and SetNativeMoments() methods
     141             : which each accept three doubles (rather than a vector) are provided and can be
     142             : repeatedly called without incurring undue overhead. The body axes force vector
     143             : can now be retrieved by calling:</p>
     144             : 
     145             : <p><tt>vFb=fgf.GetBodyForces();</tt></p>
     146             : 
     147             : <p>This method is where the bulk of the work gets done so calling it more than
     148             : once for the same set of native forces and moments should probably be avoided.
     149             : Note that the moment calculations are done here as well so they should be
     150             : retrieved after calling the GetBodyForces() method:</p>
     151             : 
     152             : <p><tt>vM=fgf.GetMoments();</tt> </p>
     153             : 
     154             : <p>As an aside, the native moments are not needed to perform the computations
     155             : correctly so, if the FGForce object is not being used to store them then an
     156             : alternate approach is to avoid the SetNativeMoments call and perform the sum</p>
     157             : 
     158             : <p><tt>vMoments+=fgf.GetMoments();</tt> <br><br>
     159             : 
     160             : after the forces have been retrieved. </p>
     161             : 
     162             : <h4>Use of the Custom Transform Type</h4>
     163             : 
     164             : <p>In cases where the native force vector is not aligned with the body, wind, or
     165             : local coordinate systems a custom transform type is provided. A vectorable
     166             : engine nozzle will be used to demonstrate its usage. Initialization is much the
     167             : same:</p>
     168             : 
     169             : <p><tt>FGForce fgf(FDMExec);</tt> <br>
     170             : <tt>fgf.SetTransformType(tCustom);</tt> <br>
     171             : <tt>fgf.SetLocation(x,y,z);</tt> </p>
     172             : 
     173             : <p>Except that here the tCustom transform type is specified and the location of
     174             : the thrust vector is used rather than the aerodynamic reference point. Thrust is
     175             : typically considered to be positive when directed aft while the body x-axis is
     176             : positive forward and, if the native system is right handed, the z-axis will be
     177             : reversed as well. These differences in sense need to be specified using by the
     178             : call: </p>
     179             : 
     180             : <p><tt>fgf.SetSense(-1,1,-1);</tt></p>
     181             : 
     182             : <p>The angles are specified by calling the method: </p>
     183             : 
     184             : <p><tt>fgf.SetAnglesToBody(pitch, roll, yaw);</tt> </p>
     185             : 
     186             : <p>in which the transform matrix is computed. Note that these angles should be
     187             : taken relative to the body system and not the local as the names might suggest.
     188             : For an aircraft with vectorable thrust, this method will need to be called
     189             : every time the nozzle angle changes, a fixed engine/nozzle installation, on the
     190             : other hand, will require it to be be called only once.</p>
     191             : 
     192             : <p>Retrieval of the computed forces and moments is done as detailed above.</p>
     193             : <br>
     194             : <pre>
     195             :     <p><i>CAVEAT: If the custom system is used to compute
     196             :     the wind-to-body transform, then the sign of the sideslip
     197             :     angle must be reversed when calling SetAnglesToBody().
     198             :     This is true because sideslip angle does not follow the right
     199             :     hand rule. Using the custom transform type this way
     200             :     should not be necessary, as it is already provided as a built
     201             :     in type (and the sign differences are correctly accounted for).</i>
     202             :     </p>
     203             : </pre>
     204             : 
     205             : <h4>Use as a Base Type</h4>
     206             : 
     207             : <p>For use as a base type, the native force and moment vector data members are
     208             : defined as protected. In this case the SetNativeForces() and SetNativeMoments()
     209             : methods need not be used and, instead, the assignments to vFn, the force vector,
     210             : and vMn, the moments, can be made directly. Otherwise, the usage is similar.<br>
     211             : <br><br></p>
     212             : 
     213             :     @author Tony Peden
     214             : */
